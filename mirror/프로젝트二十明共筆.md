## 1. 검색 관련 ¶

  * 검색할 때, 일단 가장 근접한 항목명부터 뜨고 그 다음 내용검색이 됐으면 좋겠음.
  * 어떤 검색어를 검색했을 때 그 검색어가 포함된 다른 항목이 같이 있다면 바로 검색한 항목이 떠버리는게 아니라 목록으로 전부 뜨게 하면 편할 것 같습니다. 이렇게 되면 같은 이름의 다른 항목만 뜨고 원하는 항목을 검색할 수가 없게 되어서 상당히 불편합니다.  

    * 현 리그베다위키도 상단 검색창의 왼쪽에 '찾기' 라고 되어있는 메뉴를 누르거나, 키보드로 s를 누르면 가능합니다. 그때 체크표시를 하는게 몇개 있는데, 첫번째 칸은 '내용에서만 검색' 이므로 이걸 해제해야 '제목+내용에서 검색' 기능이 되니 참고가 될 것 같네요.
    * 위키미러에서 '단어'/문서 로 찾으면 해당 단어가 들어간 항목을 찾을 수 있습니다. 단어엔 항목 이름도 되고요.
    * [FindPage](FindPage.md)에 문서 제목으로만 검색 기능이 있으면 좋겠습니다. 현재 검색어가 존재하는 항목인 경우 검색어를 포함하는 항목명 목록을 보려면 해당 항목의 주소에 `?action=likepages`를 붙이거나 위키연습장에 `[[PageList(검색어)]]`를 쓰고 미리보기를 눌러서 봐야 하는 불편함이 있습니다.  

      * 해당 항목에서 [단축키](%EB%8B%A8%EC%B6%95%ED%82%A4.md) L키를 눌러도 볼 수 있긴 한데 모바일에서 쓰기 어려운 기능이고 이 기능을 모르는 사람이 많은 듯 하네요.
      * 이미 있습니다. 제목으로 검색하기에서 검색하면 됩니다. 단, 검색어가 이미 존재하는 항목명일 땐 해당 항목으로 리다이렉트 되므로, 검색 어 이런 형식으로 띄워서 검색해야합니다.  

  * 검색기능 강화(해당 단어를 내용에 포함하는 문서 검색 등)  

    * 검색시 '검색 결과 문맥 보기'를 체크하면 가능  

  * 제목을 클릭하면 해당 단어를 링크한 모든 페이지 검색이 가능.  

    * s 키로 갈 수 있는 검색페이지에서 full text search 가능합니다.
    * 모니위키 1.1.5부터 db 인덱싱 기능이 추가되었습니다. 현재 엔하위키에서는 이 기능이 비활성화 되어있습니다.  

  * 검색어 자동 완성 기능이 있었으면 좋겠습니다. "엔"만 써도 "엔진", "엔하위키"등의 단어가 검색창 밑에 나오게요.  

    * 모니위키에 해당 기능이 있습니다  

  * 메타 기호로 검색하는 기능을 추가해 주실 수 있으신가요? 예를 들어서 경* 이라는 검색어를 입력하면 경으로 시작하는 모든 단어를 찾아낸다든지 경* T* 로 검색하면 경이나 T로 시작하는 모든 항목을 추려낼 수 있다든지. 여담으로 위키백과에는 이미 존재하는 기능입니다.   

    * [엔하위키 미러](%EC%97%94%ED%95%98%EC%9C%84%ED%82%A4%20%EB%AF%B8%EB%9F%AC.md)에 비슷한 기능이 있긴 있습니다만, 해당 글자가 들어가는 제목을 다 찾아내는지라...  

  * 영어로 쳤을 때 자동으로 한글 변환. 왜 그런 거 있잖아요. 모르고 한영 안 누르고 쳤지만, 화면에 "혹시 찾으시는 단어가 이거 아님?" 이라면서 친절히 설명해준다던가.(...)  

  * 1.01버젼 로그를 검색하는 기능이 있었으면 합니다. 이거 있으면 어떤 항목이 반달당했나 확인하기가 쉬워져요.  

    * "과거"를 누르면 이제까지의 수정내역을 볼 수 있습니다. 삭제된 항목은 해당 항목명으로 검색하면 이전 페이지가 있다고 물어보고요.   

  * 오른쪽에 있는 최근 바뀐 글의 more를 클릭하면 [http://rigvedawiki.net/r1/wiki.php/RecentChanges](RecentChanges.md) 링크가 기본 페이지인데 [http://rigvedawiki.net/r1/wiki.php/RecentChanges/more](RecentChanges/more.md) 링크를 기본 페이지로로 해줬으면 좋겠습니다. 리스트를 보고 싶어서 누르는건데 기본 페이지가 짧아서리...두 번 눌러서 보기가 좀 번거로워서요.   

    * [부하 문제가 있다고 합니다.](http://old.wikibbs.net/index.php?document_srl=5210766)   

  * 문서 삭제나 항목명 변경때 역링크를 알수 있으면 편할거 같습니다.   

    * 문서 제목을 클릭하면 역링크를 찾아줌.   

  * 현 리그베다에서 띄어쓰기를 잘못 입력 하고 입력햇을때 다시 검색해야하는 번거로움이 있는데 이것도 해결되면 좋겠습니다.  

    * 해당 기능이 존재하며 적용되어 있습니다.  

  * 모바일 페이지에서 랜덤 버튼이 있으면 좋겠습니다. 뷰어에도 있지만, 위키게 등을 왔다갔다 해야하는 일이 있어서 브라우저로 보는 일이 많아서요.  

    * 로그인 사용자라면 quicklink 기능을 이용해보시기 바랍니다.  

  * 검색 결과 추출된 목록의 배열 우선순위를 손봐주셨으면 합니다. '제목에 검색 키워드가 들어가 있는가'>'내용 중 검색 키워드가 들어가 있는가'  

  * 철자를 잘못 입력하여 검색한 경우 (예를 들어 '마이 케미컬 로맨스'를 '마이 캐미컬 로맨스'로 검색했을 경우) 나오는 검색 결과가 원래 의도와는 완전 딴판으로 추출됩니다. 네이버 처럼 'XX로 다시 검색하시겠습니까?' 정도의 기능이 있으면 좋겠네요. 현 리그베다 위키 엔진은 검색 조건이 너무 빡빡하고 어떤 알고리즘을 이용해서 결과를 추리는지 직관적이지 못합니다.  

  * 역링크에 목차, 문단으로 리다이렉트하는 항목도 나오게 했으면 좋겠습니다.

## 2. 문서 분류 관련 ¶

  * 블로그의 태그마냥 관련단어로 링크가 연결이 되어있으면 편할 듯.  

  * 표제어 분류(카테고리 생성)  

    * 모니위키에 해당 기능이 있습니다.
    * [노스모크](%EB%85%B8%EC%8A%A4%EB%AA%A8%ED%81%AC.md)처럼 위키 페이지마다 카테고리 적어주면 따로 기능 없이도 카테고리 기능을 사용할 수 있을거 같습니다.
    * 카테고리를 대체하는 키워드 기능이 모니위키 1.1.3부터 지원되기는 합니다.  

  * 각 문서에 자율적으로 태그를 달 수 있는 기능을 추가하면 어떨까요? (니코동이나 픽시브처럼) 혹은 문서들을 범주화 시킬 수 있는 기능이 있었으면 합니다. (위키피디아처럼)  

    * 그런데 이건 태그가 오히려 난립하지 않을지 걱정.
    * [TaggingSystem란게 있긴 하네요.](http://moniwiki.kldp.net/main/TaggingSystem)  

  * 지금 보는 내용과 가까운 글 보기도!!  

  * 보고 있는 문서를 본 사람들이 가장 많이 본 문서를 볼 수 있는 기능.  

  * 이 글이 링크된 글 목록, 상관있는 글끼리 링크를 더 두텁게 하고, 또 홀로 외떨어진 문서를 판별할 때 유용할 것 같습니다.  

    * 역링크 기능이 있습니다.  

    * 각 항목별로 최하단에 현재 이 글이 링크되어 있는 항목들을 글의 분량, 수정의 횟수등의 기준을 삼아서 우선순위잡아서 태그로 링크해주는 관련글 기능이 있으면 좋겟습니다.  

  * 조회수   

    * moniwiki에 PageHits 기능이 있습니다. <http://moniwiki.kldp.org/wiki.php/PageHitsMacro>  

      * 그렇다면 개편할 때 꼭 페이지 조회수를 볼 수 있도록 해주셨으면 합니다.   

  * 글 통계  

  * 글 순위 기능도 있으면 어떨까 싶습니다만.  

  * 리그베다 위키에 [외톨이 문서](%EC%99%B8%ED%86%A8%EC%9D%B4%20%EB%AC%B8%EC%84%9C.md)가 가끔 발견되고 있는데 본관에도 외톨이 문서 목록을 볼 수 있는 기능이 있으면 좋겠습니다. 미러의 고립된 항목 기능은 1년넘게 업데이트되지 않고 있어요. 또 외톨이 문서라는 것을 문서에 표시해주는 기능도 있으면 좋겠습니다.   

    * 해당 기능이 모니위키에 있습니다.  

  * 각각의 [틀](%ED%8B%80.md)이 어느 문서에 쓰였는지 알 수 있게 하는 기능이 있으면 좋겠습니다. 정확히는 `[[include(어떤항목)]]` 문법을 사용한 항목을 "어떤항목" 에서 역링크 기능과 같이 목록으로 확인 할 수 있었으면 합니다. 미디어위키에는 이 기능이 있더라고요. 예를 들면 [틀:폐선](%ED%8B%80%3A%ED%8F%90%EC%84%A0.md)이 달린 문서 목록을 볼 수 있게 되면 리그베다 위키에 등재된 폐역, 폐선된 철도, 버스 노선 항목들을 모아서 볼 수 있게 되고 [동결처리/사유](%EB%8F%99%EA%B2%B0%EC%B2%98%EB%A6%AC/%EC%82%AC%EC%9C%A0.md), [집단연구 항목의 목록 문단](%EC%A7%91%EB%8B%A8%EC%97%B0%EA%B5%AC#s-2.1.md)을 편집하는데 도움이 되잖아요. 더불어 틀을 폐기할 때에도 해당 기능이 없으면 해당 틀을 단 다른 항목에 죽은 문법 생기는 불편함이 있습니다.

      * 틀은 틀인자 기능을 쓰면 됩니다.  

  * 최근 바뀐 글에서 문서 역사에 나오는 (+n/+n)을 알 수 있었으면 좋겠어요. 사소한 수정인지 구분하기도 쉽고 대형 반달 일어날 때도 알기 쉽고…….  

    * 개발판 모니위키에서는 지원하는 기능입니다.  

  * 사진, 그림을 접을 수 있게(링크로 그 페이지에 들어가야 나오는게 아니라 블로그에서 접어 내용 접듯이) 할 수 있으면 좋겠습니다. SCP의 항목중 혐짤같은게 있을 수 있는데, 무조건 링크로 대체하기도 그렇잖아요.  

    * 전 그런 건 링크로 대체하는게 맞다고 봅니다만…페이지 내의 열기 접기 같은 건 페이지 로드 속도 면에서도 별 도움이 안될 것 같은데요. 차라리 항목 내에 직접 표시하고 싶지 않은 이미지의 경우 링크를 걸기 쉽게 하는 기능을 넣는게 낫겠죠.  

  * 목차를 접고 열 수 있게 해줬으면 좋겠습니다.  

    * 지금도 목차는 접을 수 있습니다. 기본적으로 펴져 있는거에요. 목차 옆의 삼각형을 누르면 접고 펼수 있어요.  

      * 모르는 사람이 보면 삼각형 버튼은 그냥 장식으로 보일 수 있으니 열고 닫는 버튼이란 걸 확실히 알 수 있게 하는게 좋지 않을까요. 확인해보니 PNG 이미지 파일이던데 지금같은 삼각형 이미지가 아니라 열기/닫기 같은 텍스트 이미지로 바꿔주면 될 것 같습니다.  

  * [취소선](%EC%B7%A8%EC%86%8C%EC%84%A0.md)이 쳐진 부분을 숨길 수 있는 기능도 추가해주셨으면 좋겠습니다.  

  * 본문 폰트 사이즈를 키울 수 있는 설정이 있으면 좋겠습니다.  

    * css를 수정해서 사용하거나 브라우저 자체적으로 지원합니다.
  * 최근 새로 생긴 문서 목록을 볼 수 있는 기능이 있으면 좋겠습니다. 이렇게 하면 [고지달성](%EA%B3%A0%EC%A7%80%EB%8B%AC%EC%84%B1.md) 문서를 확인하기 쉬워지잖아요.   

  * 미디어위키에는 "특수기능:필요한문서"라고 링크는 걸려 있지만 작성되지 않은 항목 목록을 볼 수 있는 기능이 있는데 리그베다위키에도 도입했으면 좋겠습니다.  

    * 모니위키에 해당 기능이 있습니다.  

  * 블로그처럼 요약글 기능이 있었으면 좋겠어요. 노래 항목이라면 되도록이면 가사가 맨 밑에 가도록 하지만, 그렇지 않을 경우 가사 밑의 항목을 보기 위해서 맨 위로 올라가서 목차를 누르거나 스크롤을 내려야 하니 불편한 것이 생기는 듯합니다. 가사를 요약글 처리해서 보면 문서 자체도 균형있게 보이지 않을까요. 또 요약글 기능이 있다면 스포일러 방지에도 도움을 주지 않을까 생각됩니다.  

  * 읽을때에 검색 기능이 있었으면 좋겠어요. 그리고 기존의 검색 기능도 그 속에서 검색이 가능하도록 하고 정확성도 더 높이면 좋겠어요. 또 역시 검색 기능에서 시간순/가나다순/정확도순 등으로 나누면 좋겠습니다.

## 6. 사용자 간 교류 관련 ¶

    * 모니위키에 해당 기능 있음.

    * 네임스페이스(이름공간)이 따로 있으면 좋겠습니다만, 현재 위키 시스템 특성상 안 될 듯 합니다.(...)
    * 쪽지는 아니지만 이메일은 쓸 수 있는듯 합니다.
  * <del>타 엔진과의 교류 관련 : [미디어위키](%EB%AF%B8%EB%94%94%EC%96%B4%EC%9C%84%ED%82%A4.md)는 공백을 _(밑줄)로 처리하지만, 이 위키는 %20을 사용하고 있어, 공백이 들어간 항목을 검색할 경우 제대로 찾지 못하는 문제가 있습니다. (이 때문에 [위키엔진](%EC%9C%84%ED%82%A4%EC%97%94%EC%A7%84.md)간의 [갈라파고스화](%EA%B0%88%EB%9D%BC%ED%8C%8C%EA%B3%A0%EC%8A%A4%ED%99%94.md)가 우려됩니다.)</del>
