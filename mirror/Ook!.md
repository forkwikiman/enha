[난해한 프로그래밍 언어](%EB%82%9C%ED%95%B4%ED%95%9C%20%ED%94%84%EB%A1%9C%EA%B7%B8%EB%9E%98%EB%B0%8D%20%EC%96%B8%EC%96%B4.md)의 한 종류.

[브레인퍽](%EB%B8%8C%EB%A0%88%EC%9D%B8%ED%8D%BD.md)을 기반으로 한 언어로, 명령어들은 모두 브레인퍽
명령어로 치환이 가능하다. 쉽게 말해 브레인퍽의 스킨... 정도라고 할까, 하지만 그 난해함은 어쩌면 브레인퍽을 능가할 수 있을 정도.

일단 Ook! 라는건 **[오랑우탄](%EC%98%A4%EB%9E%91%EC%9A%B0%ED%83%84.md) 울음 소리를 나타내는
의성어**다. 여기서 눈치깔 수 있겠지만 몇몇 [변태](%EB%B3%80%ED%83%9C.md)들에 의해 장난으로 만들어진 언어다.
그래서 모든 명령어가 Ook라는 단어로 구성되어 있다는게 특징이다(...). 제작자가 왜 오랑우탄을 주제로 삼았냐 하면, 만든 사람이
[디스크월드](%EB%94%94%EC%8A%A4%ED%81%AC%EC%9B%94%EB%93%9C.md)의 팬이라서(...) [보이지않는 대학](%EB%B3%B4%EC%9D%B4%EC%A7%80%20%EC%95%8A%EB%8A%94%20%EB%8C%80%ED%95%99.md)의 [사서](%EC%82%AC%EC%84%9C#s-1.4.3.md)가 바로 오랑우탄인데 보이지 않는 대학의 몇몇은 이 사서가
욱욱 하는 언어를 알아 듣는다(...).

제작자가 밝힌 Ook!의 디자인 컨셉은 다음과 같다

  * 오랑우탄이 읽고 쓸 수 있는 프로그래밍 언어임
  * 단순한 명령어, 기억하기 쉽고, 예를 들면 "원숭이" 처럼 긴 단어를 쓸 필요가 없도록 제작함
  * **[바나나](%EB%B0%94%EB%82%98%EB%82%98.md)는 맛있음**(...)  

기본적으로 브레인퍽 기반이기 때문에 이 언어 역시 튜링 완전하다. 그래서 Ook! 역시 이론적으로는
[컴퓨터](%EC%BB%B4%ED%93%A8%ED%84%B0.md)가 할 수 있는 일은 다 할 수 있지만... 사용하는 명령어는 다음과
같다.

  * Ook. Ook? : 포인터 증가
  * Ook? Ook. : 포인터 감소
  * Ook. Ook. : 포인터가 가리키는 바이트의 값을 증가
  * Ook! Ook! : 포인터가 가리키는 바이트의 값을 감소
  * Ook! Ook. : 포인터가 가리키는 바이트 값을 아스키 코드 문자로 출력한다
  * Ook. Ook! : 포인터가 가리키는 바이트에 아스키 코드 값을 입력한다
  * Ook! Ook? : 포인터가 가리키는 바이트의 값이 0이 되면 Ook? Ook!로 이동한다
  * Ook? Ook! : 포인터가 가리키는 바이트의 값이 0이 아니면 Ook! Ook? 로 이동한다  

...어쨌건 각각 브레인퍽의 ><+-.,[]에 대응한다. 이렇기 때문에 브레인퍽 <-> Ook! 변환기라는 프로그램도 존재한다. 본격 난해한
언어 더 난해하게 만들기.
