## 1. 개요 ¶

데이터의 소유자나 독점자 없이 누구나 손쉽게 데이터를 생산하고 인터넷에서 공유할 수 있도록 한 사용자 참여 중심의 인터넷 환경.

  

인터넷상에서 정보를 모아 보여주기만 하는 웹 1.0보다 한단계 더 발전하여 사용자가 직접 데이터를 다룰 수 있도록 데이터를 제공하는 플랫폼이
정보를 더 쉽게 공유하고 서비스 받을 수 있게 되었다.

  

사용자가 직접 참여하기 때문에 구매자들은 일방적으로 구매하라고 하는 홍보보다 사용자의 직접적인 평을 더 신뢰하고 있다고 한다.

  

[[edit](http://rigvedawiki.net/r1/wiki.php/%EC%9B%B92.0?action=edit&section=2)
]

## 2. 설명 ¶

사전적 정의가 아닌 실질적 정의는 바로 다음과 같다. **프로그램처럼 쓸 수 있는 웹 사이트**.  
옛날의 웹 사이트`[1]`은 html 문서를 작성하고, 이를 띄우는 기능밖에 없었다. 일본에 있는 개인용 홈페이지를 생각하면 쉽다. 하지만
현대의 웹 사이트는 [게시판](%EA%B2%8C%EC%8B%9C%ED%8C%90.md)이라는 가장 기본적인 형태의 문서작성 알고리즘을
내재하고 있고, 덕분에 웹 안에서 대부분의 사이트 구성을 끝낼 수 있다.

  

즉, 현존하는 대부분의 웹 사이트는 웹2.0이다. 심지어 웹 호스팅을 사도 알아서
[그누보드](%EA%B7%B8%EB%88%84%EB%B3%B4%EB%93%9C.md)정도는 깔아줄 정도.

  

아직도 웹 1.0을 이용하는 사이트는 기업 홍보용 사이트 등 극소수의 사이트밖에 없다.

  

여기까지 읽어보면 알겠지만, **상당히 낡은 개념이다**. IT발전사에서 짤막하게 언급할만한 단어일 뿐. 현재 활용하기에는 상징적인 의미를
완전히 잃었다.

## 4. [웹 3.0](%EC%9B%B9%203.0.md)? ¶

2가 있으면 당연히 3이 오듯, 웹이 발전하면 어떤 형태가 될거라는 연구가 몇몇 있다.  
웹에 [인공지능](%EC%9D%B8%EA%B3%B5%EC%A7%80%EB%8A%A5.md)이 깃들거라는 황당무계한`[2]` 예측부터
[시맨틱 웹](%EC%8B%9C%EB%A7%A8%ED%8B%B1%20%EC%9B%B9.md)을 활용할 거라는 이야기까지 다양하다.  
현실적인 방향으론 시맨틱 웹을 사용한 사이트가 다음 방향성이긴 하다.
[플리커](%ED%94%8C%EB%A6%AC%EC%BB%A4.md)
[블로그](%EB%B8%94%EB%A1%9C%EA%B7%B8.md)같은 것들을 보면 이미
[태그](%ED%83%9C%EA%B7%B8.md)정도는 다 쓰고 있으니까.
